# [창업에듀] 웹 서비스 아키텍처 및 보안 운영 구조 분석

---

## 1. 서버 구성 및 부하 분산 (Load Balancing)

우리 시스템은 **계층별 N:M 연결 구조**를 통해 서비스의 가용성과 성능을 최적화하고 있습니다. 특히 외부 사용자 접점부터 내부 로직 처리까지 다중화된 설계를 따릅니다.

### 1.1 계층별 역할 및 분업화
* **Web Server (3대 Active):**
    * **정적 리소스 처리:** HTML, CSS, JS, 이미지 등 변하지 않는 파일을 자체 디스크에서 즉시 응답.
    * **리버스 프록시:** 외부 요청을 받아 뒷단의 WAS로 전달하며, SSL 복호화(HTTPS)를 전담하여 보안 강화.
* **WAS (2대 Active):**
    * **동적 로직 처리:** Java/Spring 기반의 비즈니스 로직 수행 및 데이터베이스(DB) 연동.
    * **보안 필터링:** 유입되는 비즈니스 데이터에 대한 최종 유효성 검증 수행.

### 1.2 다중화의 목적
* **부하 분산(Load Balancing):** 트래픽을 여러 서버로 나누어 개별 서버의 과부하 방지.
* **고가용성(HA):** 특정 서버 장애 시에도 나머지 서버가 중단 없이 서비스를 지속(Failover).

## 2. Active-Standby 이중화 및 재해 복구(DR)

현재 운영 환경은 동일한 사양의 **Standby 그룹(Web 3대, WAS 2대)**을 추가로 확보하여 시스템 전체 레벨의 장애에 대비하고 있습니다.

### 2.1 이중화 운영 방식
* **Active 그룹:** 모든 사용자 트래픽을 실제로 처리하는 주 시스템.
* **Standby 그룹:** Active 그룹과 동일한 구성을 유지하며, 비상시를 대비해 대기하는 예비 시스템.
* **장애 전환(Failover):** 메인 데이터센터나 Active 그룹 전체에 마비가 발생할 경우, **GSLB(Global Server Load Balancing)**를 통해 트래픽을 즉시 Standby 그룹으로 전환하여 무중단 서비스를 보장함.

## 3. 데이터 저장 및 공유 구조 (Storage)

다중 WAS 및 이중화 환경에서도 데이터의 동일성을 유지하기 위해 공통 저장소 체계를 활용합니다.

### 3.1 NAS (Network Attached Storage)
* **역할:** 사용자 업로드 파일(이미지, 첨부파일 등)을 저장하는 **공용 창고**.
* **공유 원리:** Active와 Standby 그룹의 모든 WAS 서버가 동일한 NAS 경로를 마운트하여 사용하므로, 어떤 경로로 접속해도 동일한 파일 접근 가능.

### 3.2 DB Server (운영 및 개발 분리)
* **역할:** 사용자 정보, 게시글 등 정형화된 데이터를 관리하는 **데이터 장부**.
* **이중화:** 운영 DB 역시 실시간 복제(Replication)를 통해 Standby DB를 유지하며, 장애 시 데이터 손실 없이 복구 가능하도록 설계.
* **망 분리:** 개발 DB는 별도의 독립된 서버와 네트워크 구역에 배치하여 개발 환경의 사고가 운영 데이터에 영향을 주지 않도록 물리적으로 격리.

## 4. 보안 솔루션 및 네트워크 통제

### 4.1 다중 필터링 체계
| 솔루션 | 주요 역할 | 비유 |
| :--- | :--- | :--- |
| **WebFilter** | 웹 해킹 공격(SQL Injection, XSS 등) 차단 | 외부 침입 방지 파수꾼 |
| **EZFilter** | 개인정보(주민번호 등) 유출 방지 및 마스킹 | 내부 정보 유출 검문소 |

### 4.2 구간별 보안 정책 (DMZ 및 내부망)
1.  **외부 → DMZ (Web):** 443(HTTPS) 포트만 개방. SSL 인증서를 통한 구간 암호화 필수 적용.
2.  **DMZ (Web) → 내부망 (WAS):** 외부 IP의 직접 접근을 전면 차단하며, 오직 웹 서버 3대의 IP만 WAS 포트로 접속 허용.
3.  **내부망 (WAS) → DB/NAS:** 가장 깊은 보안망으로, 인가된 WAS 서버의 IP에 대해서만 접근을 허용하는 화이트리스트 방식 운영.
